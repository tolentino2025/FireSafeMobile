3. Objetivo da nova tela “Agenda de Inspeções”

Criar uma nova tela que exiba uma agenda de inspeções futuras e atrasadas, baseada no array schedules do InspectionContext.

Nome do arquivo e componente

Criar um novo arquivo em screens/InspectionScheduleScreen.tsx (ou similar, mas padronize o nome).

Exportar default um componente InspectionScheduleScreen.

Requisitos de UI/UX

Layout e estilo

Seguir o mesmo padrão visual de:

screens/InspectionsListScreen.tsx

screens/HomeScreen.tsx

Usar:

ScreenFlatList ou ScreenScrollView + Spacer

ThemedText / ThemedView

useTheme() para pegar fullTheme.colors

Spacing, BorderRadius de constants/theme.ts

Não alterar tema global, nem os componentes existentes, apenas criar novos estilos locais na tela de agenda.

Dados exibidos por item da agenda
Para cada InspectionSchedule (somente isActive === true), exibir em um “card”:

Título:

Nome da propriedade (propertyName) ou da empresa (companyName).

Você deve resolver companyName / propertyName usando companies e properties disponíveis dentro do InspectionContext (veja como outros screens fazem isso).

Subtítulo:

Tipo de inspeção (usar getInspectionTypeLabel(schedule.inspectionType, language)).

Frequência (getFrequencyLabel(schedule.frequency, language)).

Datas:

nextDueDate formatada de forma amigável, usando a linguagem atual (useLanguage()).

Status visual:

Se new Date(schedule.nextDueDate) < hoje: marcar como “Atrasada” (por exemplo, um chip com cor de alerta, mas respeitando o tema).

Se nextDueDate ainda no futuro: marcar como “Próxima”.

Opcional: indicar lastInspectionDate se existir (ex.: “Última inspeção: dd/mm/aaaa”).

Ordenação e filtros

Ordenar a lista de schedules por nextDueDate ascendente (mais próximas primeiro).

Criar filtros simples na parte superior da tela, semelhante aos chips de status em InspectionsListScreen.tsx, por exemplo:

“Todas”

“Atrasadas”

“Próximas”

Esses filtros DEVEM:

Manter o mesmo estilo de chips de InspectionsListScreen (usar Pressable, BorderRadius.full, Spacing.sm, cores do tema).

Comportamento ao tocar em um item

Ao tocar em um item da agenda, por enquanto:

Navegar para a tela de lista de inspeções ou de detalhe, porém sem alterar a assinatura dos tipos de navegação existentes.

Implementação sugerida (sem quebrar nada):

Ao tocar, navegar para "InspectionsList" dentro do stack de inspeções.

No futuro podemos evoluir para pré-filtrar por propriedade e tipo, mas nesse primeiro momento não modifique a API das rotas existentes.

Não alterar InspectionFormScreen nem NewInspectionScreen.
